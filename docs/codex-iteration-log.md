# Codex 迭代记录模板

本文件只作为候选人个人迭代记录的格式模板，不再直接写入任何人的真实对话记录。

每位候选人开始答题时，AI 助手必须先确认候选人名称，并为当前候选人创建独立日志文件：

```text
docs/codex-logs/candidate-<候选人明确输入的姓名>-<本机用户名>-<时间戳>-<短哈希>.md
```

同时，AI 助手需要在本机工作区创建或读取：

```text
.codex/candidate-session.json
```

该本机会话文件用于记录当前候选人的姓名和个人日志路径，避免每轮对话重新猜测写入文件。`.codex/candidate-session.json` 是本机状态，不应提交到 Git。

`候选人名称` 必须来自候选人明确输入，不能从 GitHub、Git 配置、本机用户名、邮箱、目录名或其他机器元数据推断。本机用户名、时间戳和短哈希只用于文件名去重。

## 个人日志文件格式

```md
# Codex 迭代记录

候选人名称：张三

本文件用于记录当前候选人在 Codex 客户端中迭代本项目的过程。每完成一次 Codex 迭代，都需要在文件末尾追加一条记录。

## YYYY-MM-DD HH:mm:ss ZZZ (+0800)

### 用户输入
这里记录候选人在 Codex 中输入的需求或问题。

### AI 最终输出
这里记录 Codex 本轮完成后的最终回复。

### 本轮改动文件
- src/example.tsx

### 验证结果
- pnpm lint 通过
- pnpm build 通过
- pnpm verify:answer 通过
```
